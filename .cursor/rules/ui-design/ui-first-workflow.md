## Cursor-Specific UI Development

### IDE Integration
- **Component Development**: Use Cursor's AI for component generation and optimization
- **Design System Integration**: Leverage Cursor's IntelliSense for design system components
- **Real-time Preview**: Use Cursor's live preview features for UI development
- **Code Navigation**: Navigate between components and their usage efficiently

### UI Development Workflow
1. **Design Analysis**: Use design-to-prompt workflow to analyze UI designs
2. **Component Creation**: Create UI components following UI-first methodology
3. **State Management**: Implement state management for UI components
4. **Testing**: Test UI components and user interactions
5. **Integration**: Integrate UI components with backend services

### Cursor AI Integration
- **Component Generation**: Use Cursor AI to generate UI components from specifications
- **Code Optimization**: Leverage AI for UI code optimization and best practices
- **Bug Fixing**: Use AI assistance for UI-related bug fixes
- **Documentation**: Generate component documentation using AI

## Activation Logic

### Automatic Activation
This workflow is automatically activated when:
- Working with UI-related files (*.tsx, *.jsx, *.vue, *.swift, *.kt)
- Mentioning UI, design, or user experience keywords
- Working on frontend development tasks
- Analyzing UI designs or mockups

### Manual Activation
Manually activate this workflow when:
- Starting new UI development projects
- Implementing UI-first methodology
- Conducting UI design reviews
- Training team on UI development standards

## Implementation Guidelines

### For UI Developers
1. **Follow UI First Principles**: Prioritize UI implementation before backend logic
2. **Use Templates**: Utilize all UI First templates for comprehensive development
3. **Leverage Cursor AI**: Use Cursor's AI capabilities for UI development
4. **Test Thoroughly**: Ensure all UI components are thoroughly tested

### For Designers
1. **Provide Specifications**: Use design templates to provide detailed specifications
2. **Collaborate with Developers**: Work closely with developers using shared templates
3. **Validate Implementation**: Use validation checklist to ensure design compliance
4. **Iterate Based on Feedback**: Continuously improve designs based on user feedback

**Success Criteria**: Complete UI coverage, consistent design implementation, and seamless user experience across all platforms.
